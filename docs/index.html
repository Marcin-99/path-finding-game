Path finding game documentation

Source code

  • Game created with Kotlin
  • Created 15 classes | 1096 lines of Kotlin code | Around 1000 lines of pure xml
  • You can find source code on GitHub here

Functionality

  • Application contains three Fragments: Algoritm, Game and Summary
  • At "Algoritm" fragment user can select one of three algorithms and one of two game mods, then start the game
  • At "Game" fragment user can play the game (currently 5 levels), then he can go to the summary
  • At "Summary" fragment user can see his score

Rules

  • In competetive mode game keeps track of user score. For every level user passes he gets 2000 points, but he looses 100 for every wrong field he selected and every field that he selected but shouldn't
  • Sandbox game mode allows user to create his own levels and test them
  • Path that user should select always depends on the algorithm

Classes and functions

    In app/java/com.example.path_finding_app:

  • ./MainActivity - contains state and sets up an application
  • ./fragmets - fragments with all the logic split itnto separate classes
  • ./fragmets/adapters/ViewPagerAdapter - adapter that handles the logic for switching fragments
  • ./fragmets/algorithms - classes that contain all logic for specific algorithms
  • ./fragmets/levels - easly editable classes that specify layout of all levels for every algorithm
  • ./fragmets/shared - classes and functions shared between multiple other classes
  • ./visualiser - code responsible for animations
